## 一、测评题目（Round 2 新题）

### TASK1: Coding — 泛型 EventBus

```
用 TypeScript 实现一个类型安全的泛型 EventBus（事件总线），要求：
- 支持 on(event, handler)、off(event, handler)、emit(event, data) 方法
- 支持通配符匹配，如 on('user.*', handler) 可匹配 'user.login'、'user.logout'
- 支持 once(event, handler) 方法，触发一次后自动移除
- 提供完整的 TypeScript 泛型类型约束
- 包含至少 4 个边界用例的测试代码（含通配符匹配、once、off 后不再触发、无匹配事件 emit）
```

**考察点：**
- 通配符 `*` 的语义（单层匹配 vs 多层匹配，题意暗示单层）
- `once()` 的实现方式（wrapper vs flag）
- 泛型类型推导质量
- 代码健壮性（错误处理、空安全）

## 六、待改进项（Round 3 建议）

1. **输出 token 限流**：glm-5 和 MiniMax-M2.7 触及 8192 上限，建议调高到 16384
2. **Coding 增加编译验证**：glm-4.7 的编译错误仅靠人工审查
3. **Reasoning 增加约束满足类题目**：当前题偏简单（6 会议均可安排），可增加"必须调整"类冲突
4. **翻译增加语言轮数**：如 CN→EN→JP→DE→CN，测试跨语言链式准确性
5. **引入 Code Review 任务**：给定代码找 bug，更贴近实际编码能力

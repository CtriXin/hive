# 限时秒杀功能模块 - 任务拆分方案

## 1. 子任务拆分

| ID | 描述 | 复杂度 | 预估文件 | 依赖关系 |
|----|------|--------|----------|----------|
| T1 | 秒杀商品管理 API（CRUD + 库存预扣） | 高 | 6 | 无 |
| T2 | Redis 库存扣减 Lua 脚本 + 服务层 | 高 | 4 | 无 |
| T3 | 消息队列集成（RabbitMQ/Kafka） | 中 | 3 | T2 |
| T4 | 秒杀订单异步处理器 | 高 | 5 | T2, T3 |
| T5 | 防刷策略中间件（IP 限流 + 频率控制 + 验证码） | 高 | 5 | 无 |
| T6 | 服务端时间同步 API | 低 | 2 | 无 |
| T7 | 秒杀倒计时前端组件 | 中 | 3 | T6 |
| T8 | WebSocket 结果推送服务 | 中 | 3 | T4 |
| T9 | Grafana + Prometheus 监控埋点 | 中 | 4 | 无 |

---

## 2. 并行执行组

```
Group 1 (并行): T1, T2, T5, T6, T9
       ↓
Group 2 (并行): T3, T7
       ↓
Group 3 (并行): T4, T8
```

**依赖说明**：
- T3 需要 T2 的 Redis 脚本输出作为输入
- T4 需要 T2（库存扣减）和 T3（消息队列）完成
- T7 需要 T6 的时间同步 API
- T8 需要 T4 的订单处理结果

---

## 3. 模型分配方案

| 任务 | 推荐模型 | 理由 |
|------|----------|------|
| **T1** 商品管理 API | kimi-for-coding | 核心业务逻辑复杂，需要深度分析能力设计 CRUD + 预扣逻辑 |
| **T2** Redis Lua 脚本 | kimi-for-coding | 高并发防超卖是关键难点，需要最强推理能力保证正确性 |
| **T3** 消息队列集成 | qwen3.5-plus | 标准化集成模式，编码扎实即可 |
| **T4** 订单异步处理器 | kimi-k2.5 | 稳定可靠，消息处理逻辑需要无硬伤 |
| **T5** 防刷中间件 | MiniMax-M2.7 | 策略逻辑需要详尽分析，多层限流设计复杂 |
| **T6** 时间同步 API | qwen3-coder-plus | 简单 NTP 同步逻辑，快速完成 |
| **T7** 倒计时组件 | qwen3.5-plus | 前端组件复杂度中等，编码能力足够 |
| **T8** WebSocket 推送 | MiniMax-M2.5 | 实时推送质量中上，速度较快 |
| **T9** 监控埋点 | glm-5-turbo | 标准化指标埋点，速度快且翻译好（Grafana 文档多为英文） |

---

## 4. 关键风险和缓解措施

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| **超卖问题** | 🔴 高 | T2 使用原子 Lua 脚本；增加 [DISCUSS_TRIGGER] 让两个模型交叉验证脚本逻辑 |
| **库存不一致** | 🔴 高 | T4 设计补偿事务；订单失败自动回滚预扣库存 |
| **验证码体验** | 🟡 中 | T5 采用滑动验证码 + 行为分析，避免机械式验证影响转化 |
| **WebSocket 连接数** | 🟡 中 | T8 设计连接池 + 心跳保活，单节点承载 10w+ 连接 |
| **监控指标遗漏** | 🟡 中 | T9 定义核心指标清单：QPS、成功率、库存水位、限流触发次数 |

---

## 5. 预估总耗时

```
关键路径分析（单位：秒）：

Group 1: max(130, 130, 200, 60, 60) = 200s  [T5 MiniMax-M2.7 最慢]
   ↓
Group 2: max(100, 140) = 140s  [T7 qwen3.5-plus 最慢]
   ↓
Group 3: max(100, 60) = 100s  [T4 kimi-k2.5 最慢]

─────────────────────────────
串行关键路径总计：440s ≈ 7.3 分钟
```

**并行优化建议**：
- T5（防刷）和 T2（库存）是 Group 1 的瓶颈，可考虑将 T5 换为 kimi-for-coding (130s) 与 T1 共享模型上下文
- 若开启 3 个并行 worker，实际 wall-clock 时间可压缩至约 **5 分钟**

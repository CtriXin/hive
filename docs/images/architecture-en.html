Hive v2.1.0 mainline

Hive Architecture

Current reality: CLI, MCP, and local Web all drive the same run-time control loop.

Entry Surfaces different entry points, one runtime

hive CLI
Run, resume, observe, steer, compact, restore, and inspect the latest run from the terminal.
run / resume status / watch compact / shell / web
MCP tools
Structured host entrypoints for planning, execution, reporting, run inspection, and steering.
plan_tasks run_goal / run_status submit_steering
hive web
Local browser decision surface with stop reason, next action, active runs, and model policy center.
decision-first UI policy center

Core Control Loop what actually happens inside a run

Run bootstrap + state machine
Creates `spec.json` and `state.json`, manages phases, terminal states, retries, and loop progress.
spec / state / progress
→
Planner + driver loop
Plans tasks, resumes runs, decides repair vs replan, and keeps the operator loop coherent.
plan / execute / repair / replan
→
Capability routing + provider resilience
Selects effective models, handles fallback paths, bounded retries, cooldowns, and route health.
routing / fallback / retry
→
Workers in isolated git worktrees
Each task runs in its own worktree so execution stays parallel, reviewable, and merge-safe.
parallel task execution
→
Review cascade + verification + merge
Cross-review, authority decisions, build/test verification, and progressive merge close the loop.
review / verify / merge
Layered policy + steering
Run-time policy is not global guesswork anymore. Hive resolves effective policy across layers and applies safe-point semantics for active runs.
Run > Project > Global > Default safe-point override
Discuss / advisory / memory
Worker uncertainty, advisory signals, and recall context feed into the control loop instead of staying off to the side.
Human steering
Pause, resume, request replan, and other steering actions are persisted and consumed at safe points.

Artifacts and Operator Surfaces persistent run truth, then CLI/Web views on top

.ai/runs/<run-id>
Each run persists plan, result, loop progress, worker status, score history, steering state, provider health, and model overrides.
plan / result worker-status / score-history model-overrides / provider-health
CLI operator surfaces
status, watch, compact, restore, shell, workers, and runs all read from durable run artifacts.
Web operator surfaces
